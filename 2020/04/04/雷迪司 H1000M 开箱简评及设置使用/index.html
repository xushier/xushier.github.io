雷迪司 H1000M UPS 开箱及使用

发表于 2020-04-04 更新于 2023-03-30 分类于 开箱简评 Waline： 本文字数： 1.8k 阅读时长 ≈ 7 分钟
雷迪司 H1000M UPS 简单晒单开箱及快速配置使用。

一月份（2020）配了台 Nas，本文为该 Nas 系列篇三 —— 雷迪斯 H1000M UPS 开箱及使用。关于该 Nas 后面会有文章详细介绍。

  • 篇一：乔思伯 V4 机箱选购历程及开箱简评。
  • 篇二：海韵 Focus GX550 80Plus 金牌全模电源开箱。

Nas 组装好运行了几天后，看了好多网友的 Nas 停电经历，愈发担心小姐姐们的家，于是买了这款 UPS，使用半年多了。目前为止，还算满意。

订单

型号是雷迪司 H1000M ，可带机 30 分钟款，保价 20，实际到手 249。

开箱

配件只有一根通讯线。

主机，特别重，中部两侧和上方有散热缝隙。

顶部的标签。

正面是一个屏幕以及开关机按键。

后面是四个三孔插座和一个通讯口，以及一个重置按键。

使用

基本连接使用

UPS 连接电源，理论上 4 个插座口需要用 3 个，分别是 Nas 主机、软路由以及光猫，这样可以保证断电时 Nas 和软路由仍然能够与外界通讯，以便通知断电信息。

但软路由和光猫都是两脚插头，不能接 UPS。所以用了一个三脚插排接 UPS，用作 AP 的路由也顺便能接入了。

通讯口接到 Nas 上的 USB 口，以便查看 UPS 的电压电流等状态信息，最重要的是实现断电通知关机。

配套软件安装使用

该 UPS 附带控制软件，名为 ViewPower，支持 Win、Mac、Linux 平台。安装后在浏览器打开地址可查看 UPS 状态、配置安全策略、短信通知、开关机策略等功能。


  • 本 Nas 用的 Unraid 系统属于 Linux，下载 Linux 64 位文本界面。


  • 复制下载链接。


  • 打开命令行下载：wget http://www.ladis.com.cn/down/installLADIS_Linux_text_x86_64.tar.gz解压：tar -zxf 文件


  • 安装：./文件名.bin，如果选择默认安装位置，一路按回车即可。


  • 安装完成后，打开地址：http://[NAS 本地 IP]:15178/viewpower，即可看到如上图所示的电力流。在右上角可将语言改为简体中文，默认管理员登录密码为 administrator，登录后可修改密码。



  • 负载关机和远程负载关机，感觉描述的很迷，看不懂。


  • 有邮件通知和短信通知，感觉最实用的也就是短信通知了，根据视频设置参数后测试可用。

配套软件使用体验

虽然官方配套软件，但 Unraid 系统运行在优盘，该软件安装到系统，虽安装后可正常使用，但重启后相关安装文件被清除，要使用只能重新安装。

当然，可以搭配 user scripts 来进行开机自动安装，但安装后还需要重新进行配置，并没有实际解决问题。

尝试在 Linux 虚拟机里安装该软件，但直通 USB 控制器后也检测不到 USB 信号；又尝试拆包整合 Unraid 系统，但貌似是因为安装了 Java 的原因还是不行；遂放弃。

在咨询客服后得到的答复是，断电时需要手动进入软件关机。如果真是这样，那这个软件以及 UPS 就没有什么存在和使用的必要了，显然不可能，所以客服应该也是不懂。

这么多破事，还用个 {屁 | diǎo}。

Unraid Nut 插件使用

放弃自带软件后，转用 Unraid 应用商店的 Nut 插件。


  • 傻瓜式安装，确保设置过 GitHub 相关代理且时间正确，或者有国外网络环境，在 APPS 里搜索 Nut，搜索结果第一个就是，点击下载图标即可。


  • 第一项默认为 No，设为 Yes，保存，点击 AutoDetect 即可检测到 UPS 信号。


  • 上一步设置完成并检测到信号之后，就可看到如上图的 NUT Details 即 UPS 的相关电压电流参数了。此时，网页界面最下方大概中间位置处会有一个电池图标和电量显示。


  • Unraid 的 DashBoard 界面也会多出一个 Power 板块，如图所示，显示 UPS 的状态、电量、负载及功耗。

断电上电流程

断电时，UPS 屏幕亮，并伴随间断的 “滴” 声，路由及光猫都是 UPS 供电，所以此时 Nas 还是联网的，Nas 系统给设定好的服务发送通知，之后在设定的时间内关机，关机后 UPS 也自动关机，屏幕熄灭，“滴” 声停止。

来电时，UPS “滴” 一声，屏幕会亮一会儿，光猫启动，软路由通电自启，Nas 通电自启。

PS：“滴” 声可通过自带软件关闭。

事件通知

因为停电时需要手动暂停 QB 任务来和平关闭，防止来电时出现大批量的校验甚至文件丢失，且停电时只有几分钟的时间操作，所以停电时需要有强提醒，来保证停电时可以收到消息并看到，来决定在系统自动关机时是否需要进系统进行一些操作。

但是邮件提醒只能算作辅助提醒，除了电话最好的强提醒应该就是短信提醒了。配套软件有免费短信提醒，这也是为什么花那么大精力想去用配套软件。

但既然配套软件不好用，那就只能曲线救国了。偶然看到 139 邮箱发的短信，在度娘搜索后得知 139 邮箱每月有不少免费短信额度。于是，嘿嘿，我有一个计划。

说干就干，Unraid 自带事件通知功能，如图所示，可设置通知的事件及等级等参数。

在此处设置邮箱通知，需要一个发件邮箱（A）和一个收件邮箱（B），收件邮箱为 139 邮箱。

系统产生通知后，通过 A 邮箱将消息发送给 B 邮箱，B 邮箱通过短信将消息发送给 139 邮箱注册的手机。（139 邮箱需设置收到 A 邮箱的邮件自动短信通知手机）。

Unraid 还有很多手机客户端可以进行事件通知，配置较简单。但这些 APP 一般都是国外用的多，国内没什么人用，并且需要联网，跟邮件也没什么区别。

总结

此款 UPS，相对热门 APC 等品牌更便宜，功能上也算完备，但肯定也有一些不足之处。

  • 配套软件不易用，使用 Unraid 系统可搭配插件使用，若搭配群晖等系统不一定能完美使用；
  • 插座位置在机箱后部，插拔不方便；
  • 只能用三孔插座，双孔插头要扩展插线板使用；
  • 插座间隔太近，下面两个插座不易插入；
  • 虽然没有用过，但在转化率和功率方面感觉上肯定不如大牌。

暂时就这些了。。
